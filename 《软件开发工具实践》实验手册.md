# 实验一 网络软件基础

## 一、实验目标

1. 安装并运行一个桌面网络软件（Cisco Packet Tracer）；
2. 理解C/S、B/S、P2P等不同网络软件类型与架构；
3. 通过动手搭建一个简单的局域网，直观感受网络软件通信的基本原理。

## 二、背景知识

### 2.1 三种主流网络软件架构

| 架构类型    | 全称          | 核心特征                                       | 典型例子                   |
| :---------- | :------------ | :--------------------------------------------- | :------------------------- |
| **C/S架构** | 客户端/服务器 | 需要安装专用客户端软件，客户端与服务器分工协作 | 微信、网络游戏、Outlook    |
| **B/S架构** | 浏览器/服务器 | 只需浏览器即可访问，核心功能在服务器端         | 淘宝、知乎、在线办公软件   |
| **P2P架构** | 点对点        | 每个节点既是客户端也是服务器，可直接通信       | BT下载、部分局域网通讯工具 |

### 2.2 本实验软件

本次实验选用的 **Cisco Packet Tracer** 是一款网络模拟软件，属于**桌面型C/S架构软件**——你安装的是客户端，通过它来模拟和设计网络环境。

## 三、实验环境

- **操作系统**：Windows 10/11
- **网络**：需要互联网连接以下载软件
- **软件**：Cisco Packet Tracer（免费，需注册Cisco NetAcad账号）

## 四、详细操作步骤

### 步骤一：下载Cisco Packet Tracer

1. 打开浏览器，访问Cisco Networking Academy官网：**https://www.netacad.com/**
2. 点击页面右上角的 **“Login”** 或 **“注册”** 按钮。
3. 按照提示完成免费账号的注册（需要邮箱验证）。
4. 登录后，在课程或资源页面中找到 **“Packet Tracer”** 下载入口。
5. 选择适合你操作系统的版本（如 `PacketTracer821.exe` for Windows），点击下载。

> **提示**：如果官网下载速度较慢，也可以从学校提供的教学资源平台或可信的镜像站获取安装包。

### 步骤二：安装软件

1. **找到下载好的安装文件**，**右键点击**，选择 **“以管理员身份运行”** 。
2. 在安装向导欢迎界面，点击 **“Next”** 。
3. **选择安装语言**：建议选择 **English**（英文界面有助于熟悉行业标准术语）。
4. **阅读并接受许可协议**，勾选“I accept the agreement”，点击 **“Next”** 。
5. **选择安装路径**：建议使用**默认路径**，或修改为其他**英文路径**（避免中文路径可能引发的兼容性问题）。
6. 点击 **“Next”** 继续，然后点击 **“Install”** 开始安装。
7. 等待安装进度条完成（约需3-5分钟）。
8. 点击 **“Finish”** 完成安装。

### 步骤三：首次运行与界面熟悉

1. 双击桌面上的 **Cisco Packet Tracer** 快捷方式图标，启动软件。
2. 首次启动时，可能会弹出登录窗口，使用你注册的NetAcad账号登录。
3. 进入主界面后，观察以下几个核心区域：
   - **菜单栏**（顶部）：File、Edit、Options、View、Tools等
   - **设备工具栏**（左下角）：包含路由器、交换机、PC、线缆等网络设备
   - **工作区**（中央）：用于拖放设备、搭建网络拓扑
   - **仿真控制栏**（右下角）：用于控制网络模拟的运行与调试

### 步骤四：搭建第一个局域网

#### 4.1 添加设备

1. 在左侧**设备工具栏**中，找到 **“Network Devices”** 下的 **“Switches”** （交换机图标）。
2. 将 **“2960”** 型号的交换机拖拽到中央工作区。
3. 再找到 **“End Devices”** （终端设备），将两台 **“PC”** （如 `PC-PT`）拖拽到工作区。

#### 4.2 连接设备

1. 点击设备工具栏中的 **“Connections”** （连线工具，图标为闪电或线缆）。
2. 选择 **“Copper Straight-Through”** （直通线）。
3. 依次点击 **PC1** → 选择 **FastEthernet0** 端口 → 再点击 **交换机** → 选择某个端口（如 `FastEthernet0/1`）。
4. 用同样的方法，将 **PC2** 连接到交换机的另一个端口（如 `FastEthernet0/2`）。

#### 4.3 配置IP地址

1. **双击PC1**，在弹出的窗口中选择 **“Desktop”** 选项卡。
2. 点击 **“IP Configuration”** 。
3. 在IP Address栏中输入 **`192.168.1.1`** ，Subnet Mask自动填充为 **`255.255.255.0`** 。
4. 关闭PC1的配置窗口。
5. **双击PC2**，同样进入 **Desktop → IP Configuration**。
6. 将IP Address设为 **`192.168.1.2`** ，子网掩码保持 **`255.255.255.0`** 。

#### 4.4 测试网络连通性

1. **双击PC1**，进入 **Desktop → Command Prompt** （命令行）。
2. 输入命令：**`ping 192.168.1.2`**
3. 观察返回结果：
   - 如果看到 **“Reply from 192.168.1.2: bytes=32 time<1ms TTL=128”** 等信息，说明两台PC通信成功。
   - 如果看到 **“Request timed out”** ，请检查IP地址是否配置正确、线缆连接是否可靠。

### 步骤五：观察数据包流动

1. 在Packet Tracer右下角的仿真控制栏中，点击 **“Simulation”** （仿真）模式。
2. 点击 **“Add Simple PDU”** （添加简单数据包）按钮（信封图标）。
3. 先点击 **PC1** 作为源设备，再点击 **PC2** 作为目标设备。
4. 点击右下角的 **“Auto Capture / Play”** （自动捕获/播放）按钮。
5. 观察数据包从PC1出发，经过交换机，到达PC2的完整过程。
6. 点击某个数据包事件，可以查看数据包在各层（应用层、网络层、数据链路层等）的封装细节。

## 五、常见问题与排错

| 问题现象                    | 可能原因                     | 解决方法                                                     |
| :-------------------------- | :--------------------------- | :----------------------------------------------------------- |
| 安装时提示权限不足          | 未以管理员身份运行           | 右键安装包 → “以管理员身份运行”                              |
| ping返回“Request timed out” | IP地址配置错误或线缆连接错误 | 检查PC1和PC2的IP是否在同一网段（192.168.1.x）；检查线缆类型是否选对（直通线） |
| 设备无法拖拽到工作区        | 软件卡顿或未完全加载         | 重启Packet Tracer                                            |
| 找不到设备型号              | 设备面板未展开               | 点击设备类别左侧的“+”号展开设备列表                          |

# 实验二 Web软件开发环境

## 一、实验目标

1. 安装并运行一个Web博客软件（本实验选用 **Hexo** 静态博客框架）；
2. 了解和掌握Web开发环境的基础安装与配置（Node.js、npm、Git）；
3. 理解Web软件的B/S架构特点以及静态站点与动态站点的区别。

## 二、背景知识

### 2.1 静态博客 vs 动态博客

| 类型         | 原理                                       | 特点                                 | 典型代表                     |
| :----------- | :----------------------------------------- | :----------------------------------- | :--------------------------- |
| **静态博客** | 在本地将Markdown等源文件预编译为纯HTML页面 | 加载速度快、安全性高、无需数据库     | Hexo、Hugo、VuePress、Jekyll |
| **动态博客** | 每次用户访问时由服务器端程序动态生成页面   | 功能丰富、有后台管理界面、需要数据库 | WordPress、Typecho、Halo     |

### 2.2 Web开发环境的核心组件

| 组件           | 作用                       | 本实验对应的工具               |
| :------------- | :------------------------- | :----------------------------- |
| **运行时环境** | 执行编程语言代码           | Node.js（运行JavaScript/Hexo） |
| **包管理器**   | 管理项目依赖库             | npm（Node Package Manager）    |
| **版本控制**   | 代码管理与协作             | Git                            |
| **构建工具**   | 将源文件转换为可部署的网站 | Hexo CLI                       |
| **本地服务器** | 在开发时预览网站效果       | Hexo自带的server模块           |

## 三、实验环境

- **操作系统**：Windows 10/11
- **网络**：需要互联网连接以下载软件和安装依赖包
- **所需软件**：
  - Node.js（LTS版本，含npm）
  - Git for Windows
  - Hexo CLI
  - 一个文本编辑器（VS Code或Notepad++等）

## 四、详细操作步骤

### 步骤一：安装Node.js与npm

1. 打开浏览器，访问 Node.js 官网：**https://nodejs.org/**
2. 下载 **LTS（长期支持）版本** 的Windows安装包。
3. 双击下载的安装包，启动安装向导。
4. 在安装过程中，**务必勾选“Automatically install the necessary tools”**，并确保 **“Add to PATH”** 选项被选中。
5. 其余步骤保持默认，点击“Next”直至安装完成。
6. 安装完成后，打开命令提示符，输入以下命令验证安装是否成功：
   ```
   node -v
   npm -v
   ```

### 步骤二：配置npm镜像源（可选但推荐）

在命令提示符中执行：
```
npm config set registry https://registry.npmmirror.com
```

验证配置是否生效：
```
npm config get registry
```

### 步骤三：安装Git for Windows

1. 打开浏览器，访问Git官网：**https://git-scm.com/download/win**
2. 下载适合你系统的安装包。
3. 双击安装包，启动安装向导。
4. 在安装过程中，当出现 **“Select Components”** 页面时，建议勾选 **“Git Bash Here”** 和 **“Git GUI Here”** 。
5. 在 **“Choosing the default editor”** 页面，可以选择VS Code（如果已安装）或其他编辑器。
6. 在 **“Adjusting your PATH environment”** 页面，选择 **“Git from the command line and also from 3rd-party software”** 。
7. 其余步骤保持默认，点击“Next”直至安装完成。
8. 安装完成后，**重启命令提示符**，输入以下命令验证：
   ```
   git --version
   ```
9. 配置Git的全局用户信息：
   ```
   git config --global user.name "你的姓名"
   git config --global user.email "你的邮箱@example.com"
   ```

### 步骤四：安装Hexo CLI

在命令提示符中执行：
```
npm install -g hexo-cli
```

验证安装：
```
hexo -v
```

### 步骤五：初始化博客项目

1. 选择一个你希望存放博客源码的目录，进入该目录：
   ```
   cd D:\Blog
   ```
2. 执行初始化命令创建博客项目：
   ```
   hexo init my-blog
   ```
3. 进入项目目录：
   ```
   cd my-blog
   ```
4. 安装项目依赖（如果初始化时未自动完成）：
   ```
   npm install
   ```

初始化完成后，项目目录结构如下：
```
my-blog/
├── _config.yml      # 站点主配置文件
├── package.json     # 项目依赖清单
├── scaffolds/       # 文章模板
├── source/          # 内容源文件
│   └── _posts/      # 文章存放目录
└── themes/          # 主题目录
```

### 步骤六：本地预览博客

1. 在项目目录下，启动本地预览服务：
   ```
   hexo server
   ```
2. 看到如下提示表示服务启动成功：
   ```
   INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.
   ```
3. 打开浏览器，访问 **http://localhost:4000** ，即可看到默认的Hexo博客首页。

### 步骤七：创建并发布一篇博客文章

1. 在项目目录下，使用以下命令创建一篇新文章：
   ```
   hexo new "我的第一篇博客"
   ```
2. 用文本编辑器打开生成的Markdown文件，修改Front-matter信息和正文内容。
3. 保存文件后，重新生成静态页面：
   ```
   hexo generate
   ```
4. 再次启动本地预览服务查看效果：
   ```
   hexo server
   ```

### 步骤八：更换主题（可选）

以安装流行的NexT主题为例：

1. 在项目目录下，克隆NexT主题：
   ```
   git clone https://github.com/theme-next/hexo-theme-next themes/next
   ```
2. 打开项目根目录下的 `_config.yml` 文件，找到 `theme` 字段，将其修改为：
   ```yaml
   theme: next
   ```
3. 重新生成并预览。

## 五、常见问题与排错

| 问题现象                           | 可能原因                        | 解决方法                               |
| :--------------------------------- | :------------------------------ | :------------------------------------- |
| `'node' 不是内部或外部命令`        | Node.js未正确安装或未添加到PATH | 重新安装Node.js，确保勾选“Add to PATH” |
| `npm install` 速度极慢             | 默认使用国外镜像源              | 切换为国内镜像源                       |
| `hexo -v` 报错                     | Hexo CLI未全局安装              | 执行 `npm install -g hexo-cli`         |
| `hexo server` 启动后浏览器无法访问 | 端口4000被占用                  | 更换端口：`hexo server -p 4001`        |



| 问题现象                                | 可能原因                      | 解决方法                                                     |
| :-------------------------------------- | :---------------------------- | :----------------------------------------------------------- |
| 安装VS Code后无法在命令行使用`code`命令 | 安装时未勾选“添加到PATH”      | 重新运行安装程序，勾选“添加到PATH”；或手动将VS Code安装目录添加到系统环境变量 |
| 扩展面板搜索不到插件                    | 网络问题或搜索关键词不准确    | 检查网络连接；尝试使用准确的插件全名（如“Markdown All in One”）搜索 |
| 预览窗口不显示或显示异常                | 插件未正确安装或未启用        | 检查扩展面板中插件是否已启用；重启VS Code                    |
| 预览中图片无法显示                      | 图片路径错误                  | 检查图片路径是否正确，建议使用相对路径                       |
| `[TOC]`不生成目录                       | 未安装Markdown All in One插件 | 安装并启用Markdown All in One插件                            |

# 实验三 代码编辑器

## 一、实验目标

1. 安装Visual Studio Code（VS Code）文本代码编辑器；
2. 安装和配置Markdown相关插件，掌握代码编辑器插件的基本概念；
3. 学习Markdown基础语法，使用VS Code完成Markdown文档的编辑和渲染任务。

## 二、背景知识

### 2.1 代码编辑器 vs 集成开发环境（IDE）

| 类型                    | 特点                                                | 典型代表                         |
| :---------------------- | :-------------------------------------------------- | :------------------------------- |
| **代码编辑器**          | 轻量级、启动快、以编辑为核心功能、通过插件扩展能力  | VS Code、Sublime Text、Notepad++ |
| **集成开发环境（IDE）** | 功能全面、集成了编译/调试/构建/版本控制等全套工具链 | IntelliJ IDEA、PyCharm、Eclipse  |

### 2.2 插件（Extension）的概念

插件是VS Code最核心的扩展机制。通过安装不同的插件，可以为编辑器增加语法高亮、代码补全、格式化、预览、调试等新功能。

### 2.3 Markdown简介

Markdown是一种轻量级标记语言，通过简单的纯文本语法实现结构化文档的编写，并支持转换为HTML、PDF等格式。

**Markdown的核心特点**：
- **简洁性**：仅需少量符号（如`#`、`*`、`-`）即可完成排版
- **可读性**：未渲染的纯文本仍然清晰易读
- **兼容性**：支持GitHub、各大博客平台、笔记工具等场景

## 三、实验环境

- **操作系统**：Windows 10/11
- **网络**：需要互联网连接以下载软件和插件
- **所需软件**：
  - Visual Studio Code
  - VS Code插件：Markdown All in One、Markdown Preview Enhanced等

## 四、详细操作步骤

### 步骤一：安装Visual Studio Code

1. 打开浏览器，访问VS Code官网：**https://code.visualstudio.com/**
2. 点击首页的 **“Download for Windows”** 按钮，下载最新版本的安装包。
3. 双击下载的安装包，启动安装向导。
4. 在 **“选择附加任务”** 页面，**建议勾选以下选项**：
   - ✅ 创建桌面快捷方式
   - ✅ 将“通过Code打开”操作添加到Windows资源管理器文件上下文菜单
   - ✅ 将“通过Code打开”操作添加到Windows资源管理器目录上下文菜单
   - ✅ 将Code注册为受支持的文件类型的编辑器
   - ✅ 添加到PATH（**必须勾选**）
5. 点击 **“安装”** 开始安装。

### 步骤二：首次运行与界面熟悉

1. 双击桌面上的VS Code快捷方式图标启动。
2. 熟悉VS Code的主界面布局：
   - **活动栏**（左侧最窄的竖条）：包含资源管理器、搜索、源代码管理、运行和调试、扩展等图标
   - **侧边栏**（活动栏右侧）：显示当前活动栏工具的具体内容
   - **编辑器区域**（中央）：用于编辑文件的主要区域

# 实验四：Web站点构建——操作说明书


## 一、实验目标

1. 使用静态站点构建工具（Hexo）构建一个静态Web博客站点；
2. 了解和掌握Web应用开发的基本要素与流程：环境准备 → 项目初始化 → 内容编写 → 主题配置 → 本地预览 → 部署上线；
3. 理解静态站点生成器（SSG）的工作原理及其与动态站点的区别。


## 二、背景知识

### 2.1 什么是静态站点生成器（SSG）？

静态站点生成器（Static Site Generator, SSG）是一类工具，它将**Markdown等源文件**和**模板文件**作为输入，**预先编译**成完整的HTML、CSS、JavaScript静态网页。它的核心理念是“**提前做好**”，而非“**现场制作**”。

**静态站点 vs 动态站点：**

| 对比维度         | 静态站点（SSG）        | 动态站点（如WordPress）         |
| :--------------- | :--------------------- | :------------------------------ |
| **页面生成时机** | 构建时预先生成所有页面 | 用户访问时由服务器实时生成      |
| **数据库**       | 不需要                 | 通常需要                        |
| **访问速度**     | 极快（纯静态文件）     | 较慢（需执行后端代码+查数据库） |
| **安全性**       | 高（无后端攻击面）     | 需额外防护                      |
| **部署方式**     | 上传静态文件即可       | 需配置服务器环境和数据库        |
| **适用场景**     | 博客、文档站、个人网站 | 电商、社交、CMS                 |

### 2.2 为什么选择Hexo？

Hexo是一个基于Node.js的快速、简洁且高效的博客框架。它使用Markdown解析文章，在几秒内即可利用靓丽的主题生成静态网页。Hexo拥有**400+主题**和丰富的插件生态，适合技术博客和个人网站。

### 2.3 Web应用开发的基本流程

本次实验将完整覆盖Web应用开发的**六个核心环节**：

> **环境准备** → **项目初始化** → **内容编写** → **主题定制** → **本地预览** → **部署上线**


## 三、实验环境

- **操作系统**：Windows 10/11（本说明书以Windows为例）
- **网络**：需要互联网连接
- **前置条件**：已完成实验二（Node.js、npm、Git已安装并配置）
- **所需账号**：GitHub账号（用于免费部署）

# 实验五 虚拟机安装与使用

## 一、实验目标

1. 使用虚拟机工具（VirtualBox）安装Linux系统（Ubuntu Server）；
2. 通过克隆技术构建一个多机虚拟化环境；
3. 了解和掌握基于虚拟机的开发环境的基本构建方法和配置。

## 二、背景知识

### 2.1 什么是虚拟化技术？

虚拟化技术允许在一台物理计算机（宿主机）上同时运行多个独立的操作系统（虚拟机），每个虚拟机都拥有自己独立的CPU、内存、硬盘和网络资源。

### 2.2 虚拟机网络的三种核心模式

| 网络模式                | 特点                               | 典型用途               |
| :---------------------- | :--------------------------------- | :--------------------- |
| **NAT**                 | 虚拟机通过宿主机上网，外部不可直达 | 日常上网、安装软件     |
| **桥接（Bridged）**     | 虚拟机直接接入宿主机所在局域网     | 需要被其他设备直接访问 |
| **仅主机（Host-Only）** | 宿主机与虚拟机形成隔离私网         | 搭建内网互通实验环境   |

本实验将采用 **“NAT + 仅主机”双网卡**方案。

## 三、实验环境

- **操作系统**：Windows 10/11
- **网络**：需要互联网连接
- **硬件要求**：CPU支持VT-x/AMD-V虚拟化技术；内存≥8GB；硬盘剩余空间≥50GB
- **所需软件**：
  - Oracle VirtualBox
  - Ubuntu Server 22.04 LTS ISO镜像

## 四、详细操作步骤

### 步骤一：下载与安装VirtualBox

1. 访问VirtualBox官网：**https://www.virtualbox.org/**
2. 下载Windows版本安装包并安装
3. 安装完成后**重启电脑**

### 步骤二：下载Ubuntu Server ISO镜像

访问Ubuntu官网：**https://ubuntu.com/download/server** ，下载22.04 LTS版本。

### 步骤三：创建第一台虚拟机（模板机）

#### 3.1 新建虚拟机

- **名称**：`ubuntu-template`
- **ISO映像**：选择下载的Ubuntu Server ISO
- **类型**：Linux，**版本**：Ubuntu (64-bit)

#### 3.2 分配硬件资源

- **内存**：2048 MB或以上
- **硬盘**：VDI格式，动态分配，20 GB或以上

#### 3.3 配置虚拟机网络（双网卡）

| 适配器  | 连接方式                  | 用途           |
| :------ | :------------------------ | :------------- |
| 适配器1 | NAT                       | 上网和下载软件 |
| 适配器2 | 仅主机（Host-Only）适配器 | 多机互联       |

### 步骤四：安装Ubuntu Server

1. 启动虚拟机，选择 **“Try or Install Ubuntu Server”**
2. 选择语言和键盘布局
3. 网络配置保持默认DHCP

7. 等待安装完成后重启

### 步骤五：配置静态IP地址

登录系统后，编辑Netplan配置文件：
```bash
sudo nano /etc/netplan/50-cloud-init.yaml
```

修改为：
```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:           # NAT网卡
      dhcp4: yes
    enp0s8:           # 仅主机网卡
      dhcp4: no
      addresses:
        - 192.168.56.101/24
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]
```

应用配置：
```bash
sudo netplan apply
```

### 步骤六：配置主机名与hosts解析

```bash
sudo hostnamectl set-hostname template
sudo nano /etc/hosts
```

添加：
```
192.168.56.101 template
192.168.56.102 node1
192.168.56.103 node2
192.168.56.104 node3
```

### 步骤七：克隆虚拟机

1. 关闭模板机：`sudo shutdown now`
2. 在VirtualBox中右键点击模板机 → **“克隆”**
3. 选择 **“完全克隆”**
4. 分别克隆出 `node1`、`node2`、`node3`

### 步骤八：修改克隆机的主机名和IP

对每台克隆机：
```bash
sudo hostnamectl set-hostname node1  # 分别改为node1/node2/node3
sudo nano /etc/netplan/50-cloud-init.yaml  # 修改IP为192.168.56.102/103/104
sudo netplan apply
```

### 步骤九：验证多机环境

在node1上测试：
```bash
ping -c 3 8.8.8.8      # 测试外网
```

## 五、常见问题与排错

### 2.1 Linux命令行基础

- **命令提示符**：`用户名@主机名:当前目录$`（`$`表示普通用户，`#`表示root用户）
- **Tab补全**：输入命令或路径时按Tab键自动补全
- **快捷键**：`Ctrl+C`终止当前命令，`Ctrl+D`退出终端

### 2.2 用户与权限

- **普通用户**：受限权限用户，执行系统级操作时需要`sudo`提权
- **用户组**：将多个用户归类，方便统一管理权限

### 2.3 环境变量

- `PATH`：指定可执行文件的搜索路径
- `HOME`：当前用户的主目录路径
- `USER`：当前用户名
- `SHELL`：当前使用的Shell类型

# 实验七 远程登录管理

## 一、实验目标

1. 安装和配置OpenSSH服务，实现多机之间的远程登录管理；
2. 配置SSH密钥认证，实现免密登录；
3. 了解多机开发环境下的账户管理方法；
4. 掌握使用SCP和rsync进行跨机文件传输与同步。

## 二、背景知识

### 2.1 什么是SSH？

SSH（Secure Shell）是一种**加密网络协议**，用于在不安全的网络中安全地远程登录和执行命令。SSH默认使用**22端口**。

### 2.2 SSH的两种认证方式

| 认证方式     | 原理                    | 安全性 | 适用场景             |
| :----------- | :---------------------- | :----- | :------------------- |
| **密码认证** | 每次登录输入密码        | 较低   | 临时访问             |
| **密钥认证** | 使用公钥/私钥对验证身份 | 高     | 日常管理、自动化脚本 |

### 2.3 文件传输工具

| 工具      | 特点                       | 适用场景             |
| :-------- | :------------------------- | :------------------- |
| **SCP**   | 基于SSH的简单复制          | 一次性、少量文件传输 |
| **rsync** | 增量同步，只传输变化的部分 | 目录同步、定期备份   |

## 三、实验环境

- **操作系统**：Ubuntu Server 22.04 LTS
- **网络**：所有虚拟机处于同一Host-Only网络（`192.168.56.0/24`）
- **前置条件**：已完成实验五，拥有至少两台Ubuntu虚拟机（node1和node2）

## 四、详细操作步骤

### 第一部分：SSH服务安装与基础配置

#### 步骤一：检查SSH服务状态

```bash
sudo systemctl status ssh        # 检查服务状态
sudo apt install openssh-server -y  # 如未安装
sudo systemctl enable ssh         # 设置开机自启
```

#### 步骤二：配置SSH服务器

编辑 `/etc/ssh/sshd_config`：
```bash
sudo nano /etc/ssh/sshd_config
```

关键配置项：
```bash
# Port 22                    # 默认端口
PermitRootLogin no            # 禁止root直接登录
PubkeyAuthentication yes      # 允许公钥认证
PasswordAuthentication yes    # 允许密码认证
```

重启服务：
```bash
sudo systemctl restart ssh
```

### 第二部分：SSH密钥认证配置

#### 步骤三：生成SSH密钥对（在node1上）

```bash
ssh-keygen -t ed25519 -C "你的邮箱@example.com"
```

生成后查看：
```bash
ls ~/.ssh/
# id_ed25519（私钥） 和 id_ed25519.pub（公钥）
```

#### 步骤四：将公钥复制到远程服务器

**方法一（推荐）** ：
```bash
ssh-copy-id dev@192.168.56.102
```

**方法二（手动）** ：
```bash
cat ~/.ssh/id_ed25519.pub
# 复制内容，在node2上执行：
mkdir -p ~/.ssh
echo "粘贴公钥内容" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

#### 步骤五：测试免密登录

```bash
ssh dev@192.168.56.102
# 应无需密码直接登录
```

### 第三部分：文件传输

#### 步骤六：使用SCP传输文件

```bash
# 上传文件
scp /home/dev/test.txt dev@192.168.56.102:/home/dev/

# 上传目录
scp -r /home/dev/project/ dev@192.168.56.102:/home/dev/

# 下载文件
scp dev@192.168.56.102:/home/dev/log.txt ./

# 下载目录
scp -r dev@192.168.56.102:/home/dev/backup/ ./
```

#### 步骤七：使用rsync同步文件

```bash
# 安装rsync
sudo apt install rsync -y

# 本地同步
rsync -avz /home/dev/source/ /home/dev/backup/

# 推送到远程
rsync -avz -e ssh /home/dev/project/ dev@192.168.56.102:/home/dev/project/

# 从远程拉取
rsync -avz -e ssh dev@192.168.56.102:/home/dev/logs/ ./logs/

# 保持完全一致（--delete会删除目标中多余的文件）
rsync -avz --delete -e ssh /home/dev/source/ dev@192.168.56.102:/home/dev/dest/

# 排除特定文件
rsync -avz --exclude='*.log' --exclude='tmp/' -e ssh /home/dev/ dev@192.168.56.102:/home/dev/
```

## 五、常见问题与排错

| 问题现象                   | 可能原因             | 解决方法                              |
| :------------------------- | :------------------- | :------------------------------------ |
| `ssh: command not found`   | SSH客户端未安装      | `sudo apt install openssh-client`     |
| `Connection refused`       | SSH服务未运行        | `sudo systemctl start ssh`            |
| `Permission denied`        | 密码错误或密钥未配置 | 检查用户名密码；检查`authorized_keys` |
| 配置免密后仍需密码         | 文件权限不对         | `chmod 600 ~/.ssh/authorized_keys`    |
| `rsync: command not found` | 远程未安装rsync      | `sudo apt install rsync -y`           |

## 四、详细操作步骤

### 第一部分：GitHub Pages自动化部署

#### 步骤一：准备工作

确认Hexo博客正常运行：
```bash
cd ~/my-blog
hexo server
```

确认Git配置：
```bash
git config --global user.name
git config --global user.email
```

#### 步骤二：创建两个GitHub仓库

| 仓库     | 名称               | 可见性  | 用途         |
| :------- | :----------------- | :------ | :----------- |
| 源码仓库 | `my-blog`          | Private | 存放Hexo源码 |
| 部署仓库 | `用户名.github.io` | Public  | 托管静态页面 |

#### 步骤三：将本地项目关联到源码仓库

```bash
cd ~/my-blog
git init
git remote add origin https://github.com/你的用户名/my-blog.git
```

创建`.gitignore`：
```
node_modules/
.DS_Store
public/
.deploy_git/
```

提交并推送：
```bash
git add .
git commit -m "初始化Hexo博客源码"
git branch -M main
git push -u origin main
```

#### 步骤四：配置GitHub Actions

创建工作流目录和文件：
```bash
mkdir -p .github/workflows
nano .github/workflows/deploy.yml
```

配置文件内容（**请修改用户名**）：
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: 检出源码
        uses: actions/checkout@v3

      - name: 安装Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'

      - name: 缓存依赖
        uses: actions/cache@v3
        with:
          path: ~/.npm
          key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}

      - name: 安装依赖
        run: npm install

      - name: 生成静态文件
        run: npx hexo generate

      - name: 创建.nojekyll文件
        run: echo "" > public/.nojekyll

      - name: 部署到GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.PERSONAL_TOKEN }}
          publish_dir: ./public
          publish_branch: main
          external_repository: 你的用户名/你的用户名.github.io
```

提交并推送：
```bash
git add .github/workflows/deploy.yml
git commit -m "添加GitHub Actions自动部署"
git push
```

#### 步骤五：生成并配置Personal Access Token

1. GitHub → Settings → Developer settings → Personal access tokens
2. Generate new token，勾选`repo`权限
3. 复制Token
4. 在源码仓库 → Settings → Secrets → Actions → New repository secret
   - Name: `PERSONAL_TOKEN`
   - Secret: 粘贴Token

#### 步骤六：配置GitHub Pages

部署仓库 → Settings → Pages → Branch: `main` → Save

访问：**https://你的用户名.github.io**

### 第二部分：部署到云服务器（虚拟机）

#### 步骤七：安装和配置Nginx

```nginx
server {
```

#### 步骤八：使用rsync部署

在本地生成静态文件：
```bash
cd ~/my-blog
```

```bash
rsync -avz -e ssh --delete public/ dev@192.168.56.101:/var/www/blog/
```

## 五、常见问题与排错
