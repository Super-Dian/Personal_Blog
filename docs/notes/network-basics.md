# 网络软件基础

## 实验目的

通过 Cisco Packet Tracer 认识常见网络软件的形态，理解 C/S 架构、B/S 架构和本地仿真软件的区别，并完成一个简单局域网的搭建与连通性验证。

## 实验环境

| 项目 | 内容 |
| --- | --- |
| 操作系统 | Windows 10/11 |
| 实验软件 | Cisco Packet Tracer |
| 网络要求 | 可访问 Cisco NetAcad 下载页面 |
| 实验对象 | 两台 PC 与一台交换机组成的局域网 |

## 实验过程

### 安装并启动 Packet Tracer

从 Cisco NetAcad 下载 Packet Tracer 安装包，完成安装后启动软件。首次进入时需要熟悉设备栏、工作区、连线工具和仿真模式。

### 搭建第一个局域网

1. 在工作区添加两台 PC 和一台交换机。
2. 使用 Copper Straight-Through 线缆连接 PC 与交换机。
3. 分别配置两台 PC 的 IPv4 地址。
4. 使用 `ping` 验证两台主机之间是否连通。

示例地址规划：

| 设备 | IP 地址 | 子网掩码 |
| --- | --- | --- |
| PC0 | `192.168.1.10` | `255.255.255.0` |
| PC1 | `192.168.1.11` | `255.255.255.0` |

### 观察数据包流动

切换到 Simulation 模式，重新执行 `ping`，观察 ICMP 请求和响应数据包在 PC 与交换机之间的传递过程。

## 实验结果

两台 PC 能够互相 `ping` 通，说明局域网连接和 IP 配置正确。仿真模式中可以观察到 ICMP 数据包从源主机发出，经交换机转发后到达目标主机，并返回响应。

## 问题与解决

| 问题 | 原因 | 解决方法 |
| --- | --- | --- |
| `ping` 不通 | IP 地址不在同一网段 | 检查 IP 和子网掩码 |
| 端口未亮起 | 线缆类型或连接端口错误 | 重新选择正确线缆连接 |
| 软件无法登录 | 网络或账号问题 | 检查网络，确认 NetAcad 账号可用 |

## 实验总结

本实验帮助我们理解了网络软件的基本使用方式，也通过可视化方式认识了局域网通信过程。Packet Tracer 的仿真模式让抽象的数据包传输变得直观，为后续理解服务器访问、SSH 远程登录和网站部署打下基础。
