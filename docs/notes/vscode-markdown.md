# 实验三：代码编辑器

## 一、实验目标

1. 安装Visual Studio Code（VS Code）文本代码编辑器；
2. 配置VS Code远程SSH开发环境；
3. 学习Markdown基础语法。

## 二、实验环境

- **本地操作系统**：Windows 11
- **远程服务器**：Ubuntu 24.04.4 LTS
- **代码编辑器**：Visual Studio Code 1.89.0
- **远程开发插件**：Remote - SSH

## 三、实验过程

### 步骤一：安装Visual Studio Code

1. 访问VS Code官网：https://code.visualstudio.com/
2. 下载Windows版本安装包
3. 运行安装程序，勾选以下选项：
   - ✅ 创建桌面快捷方式
   - ✅ 将"通过Code打开"操作添加到Windows资源管理器文件上下文菜单
   - ✅ 将"通过Code打开"操作添加到Windows资源管理器目录上下文菜单
   - ✅ 将Code注册为受支持的文件类型的编辑器
   - ✅ 添加到PATH（必须勾选）

**安装完成后验证**：

```bash
code --version
# 输出：1.89.0 (user setup)
```

### 步骤二：配置远程SSH开发

**安装Remote - SSH插件**：

1. 打开VS Code扩展面板（`Ctrl+Shift+X`）
2. 搜索 `Remote - SSH`
3. 点击安装

**配置SSH连接**：

1. 按 `Ctrl+Shift+P` 打开命令面板
2. 输入 `Remote-SSH: Connect to Host`
3. 输入服务器地址：`root@10.1.107.20`
4. 选择Linux平台
5. 输入密码连接

**SSH配置文件**（`~/.ssh/config`）：

```
Host my-server
    HostName 10.1.107.20
    User root
    Port 22
```

**连接成功后**：

- 左下角显示绿色SSH图标
- 可以直接在VS Code中编辑服务器上的文件
- 终端自动连接到远程服务器

## 六、出现问题

### 问题1：SSH连接失败

**现象**：无法连接到远程服务器

**原因**：SSH服务未运行或端口被防火墙阻止

**解决方案**：

```bash
sudo systemctl status ssh
sudo systemctl start ssh
sudo ufw allow 22
```

### 问题2：Markdown预览不显示图片

**现象**：在Markdown中插入的图片在预览中不显示

**原因**：图片路径错误或图片文件不存在

**解决方案**：

```bash
# 使用相对路径
![图片描述](./images/example.png)

# 确保图片文件存在
ls -la ./images/
```
