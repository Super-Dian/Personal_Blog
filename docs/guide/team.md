# 小组分工与成果说明

## 项目目标

本项目以小组为单位完成一个课程实践静态站点，内容覆盖 Web 开发环境、代码编辑、版本管理、Linux 虚拟机、远程登录和软件部署等实验过程。

## 成果清单

| 成果 | 状态 | 说明 |
| --- | --- | --- |
| 静态站点源码 | 已完成 | 基于 VitePress 构建 |
| GitHub 仓库 | 已完成 | 用于代码版本管理 |
| GitHub Pages 部署 | 已完成 | 推送 `main` 后自动部署 |
| 虚拟机部署记录 | 待补充截图 | 需要补充实际 Nginx 访问截图 |
| 实验报告页面 | 已补充结构 | 需要按实际操作补充截图 |
| 最终 PDF 报告 | 待导出 | 可由 Markdown 页面整理导出 |

## 小组分工

| 成员 | 分工 | 具体内容 |

| 成员 A | 项目搭建 | 初始化 VitePress、配置目录结构和主题 |
| 成员 B | Web 内容 | 编写 Web 开发环境与站点构建实验 |

> 提交前请将成员 A-G 替换为真实姓名，并按实际贡献调整内容。

## 展示要点

答辩或展示时建议按以下顺序说明：

1. 介绍项目目标和仓库地址。
2. 展示 GitHub Pages 在线站点。
3. 展示实验页面结构。
4. 展示虚拟机 IP、SSH 登录和 Nginx 访问。
5. 说明 GitHub Actions 自动部署流程。
6. 说明小组分工和每个人的贡献。
